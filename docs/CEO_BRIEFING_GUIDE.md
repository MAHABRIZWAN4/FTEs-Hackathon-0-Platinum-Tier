# CEO Briefing System - Platinum Tier

## Overview

The Platinum Tier AI Employee automatically generates comprehensive weekly CEO briefings every Sunday at 9am UTC. These briefings provide executive-level insights into operations, finances, and system health.

## How It Works

### Automatic Generation (Every Sunday 9am UTC)

**Workflow:** `.github/workflows/ceo-briefing.yml`

1. **Data Collection**
   - Reads completed tasks from `AI_Employee_Vault/Done/` (last 7 days)
   - Reads financial data from `AI_Employee_Vault/Accounting/Current_Month.md`
   - Reads system health from `AI_Employee_Vault/Logs/system_health.md`

2. **Analysis**
   - Counts completed tasks by type
   - Extracts email and social media metrics
   - Parses financial data (revenue, expenses, profit)
   - Evaluates system health status
   - Identifies alerts and issues

3. **Generation**
   - Uses Claude Sonnet 4.6 to generate executive summary
   - Professional business language
   - Concise but informative
   - Highlights achievements and concerns

4. **Delivery**
   - Saves to `AI_Employee_Vault/Briefings/YYYY-MM-DD_CEO_Briefing.md`
   - Commits to repository
   - Available for review

### Manual Generation

Generate a briefing anytime:

```bash
python scripts/platinum_ceo_briefing.py
```

## Data Sources

### 1. Completed Tasks (`AI_Employee_Vault/Done/`)

**What it provides:**
- Total tasks completed in last 7 days
- Task breakdown by type
- Email handling metrics
- Social media post metrics
- Approval processing metrics

**Format:** JSON files with task metadata

**Example:**
```json
{
  "type": "email_draft",
  "status": "completed",
  "created_at": "2026-03-14T10:30:00",
  "completed_at": "2026-03-14T11:00:00"
}
```

### 2. Accounting Data (`AI_Employee_Vault/Accounting/Current_Month.md`)

**What it provides:**
- Revenue for the week
- Expenses for the week
- Profit/loss calculation
- Transaction summary

**Format:** Markdown with key-value pairs

**Example:**
```markdown
# March 2026 Accounting

Revenue: $10,000
Expenses: $3,500
Profit: $6,500
```

### 3. System Health (`AI_Employee_Vault/Logs/system_health.md`)

**What it provides:**
- Overall system status
- Error counts
- Pending approvals
- Workflow status
- Active alerts

**Format:** Markdown health report (generated by watchdog)

## Customizing the Briefing

### Change Generation Schedule

Edit `.github/workflows/ceo-briefing.yml`:

```yaml
on:
  schedule:
    - cron: '0 9 * * 0'  # Every Sunday at 9am UTC
    # Change to:
    - cron: '0 9 * * 1'  # Every Monday at 9am UTC
    # Or:
    - cron: '0 9 1 * *'  # First day of every month at 9am UTC
```

### Add Custom Data Sources

## Troubleshooting

### Briefing Not Generated

**Check workflow status:**
```bash
gh run list --workflow=ceo-briefing.yml --limit 5
```

**View workflow logs:**
```bash
gh run view --log
```

**Common issues:**
- ANTHROPIC_API_KEY not set in GitHub Secrets
- Workflow not enabled
- Cron schedule not triggered yet

**Fix:**
```bash
# Manually trigger workflow
gh workflow run ceo-briefing.yml

# Check secrets
gh secret list
```

### Empty or Incomplete Briefing

**Check data sources:**
```bash
# Check if Done folder has tasks
ls AI_Employee_Vault/Done/

# Check if accounting file exists
cat AI_Employee_Vault/Accounting/Current_Month.md

# Check if health file exists
cat AI_Employee_Vault/Logs/system_health.md
```

**Common issues:**
- No completed tasks in last 7 days
- Accounting file missing or empty
- Health file not generated

**Fix:**
- Complete some tasks first
- Create accounting file manually
- Run watchdog to generate health report

### Claude API Error

**Check API key:**
```bash
# Verify secret is set
gh secret list | grep ANTHROPIC_API_KEY
```

**Check API quota:**
- Visit console.anthropic.com
- Check usage and limits

**Fallback:**
The script automatically uses fallback generation if Claude API fails.

### Briefing Not Committed

**Check git status:**
```bash
git status
```

**Check workflow permissions:**
- Go to Settings → Actions → General
- Ensure "Read and write permissions" is enabled

**Manual commit:**
```bash
git add AI_Employee_Vault/Briefings/
git commit -m "Add CEO briefing"
git push
```
